## 1. 代理不通导致 git submodule 无法拉取

### 现象

```text
fatal: unable to access 'https://github.com/...': 
Failed to connect to 127.0.0.1 port 7890: Connection refused
```

### 根因

全局 git 配置了 `http.proxy = http://127.0.0.1:7890`，但 Clash for Windows 只有内核服务 (`clash-core-service`) 在运行，GUI 未启动。内核服务本身不读取 `config.yaml`，它需要 GUI 通过内部 API (`127.0.0.1:53000`) 推送完整配置。GUI 没有运行，所以配置是空的，端口 7890 上无进程监听。

### 修复

手动启动 Clash for Windows GUI：

```bash
/home/wuta/Downloads/Clash\ for\ Windows-0.20.4-x64-linux/cfw --no-sandbox &
```

GUI 启动后会自动将完整配置（GLaDOS profile，含 ss 节点、代理组、规则）通过 API 推送到内核服务，端口 7890 恢复监听。

验证：

```bash
curl -x http://127.0.0.1:7890 -s -o /dev/null -w "%{http_code}" https://github.com
# 返回 200
```

### 后续便利化

创建了桌面快捷方式 `/home/wuta/Desktop/clash-for-windows.desktop`，方便以后双击启动。

## 2. CMake 版本过低

### 现象

```text
CMake Error: CMake 3.24 or higher is required. You are running version 3.22.1
```

kiss_icp 的子依赖 [Sophus](https://github.com/strasdat/Sophus) 要求在 CMake 3.24+ 下构建，Ubuntu 22.04 官方源最高只到 3.22。

### 修复

通过 pip 安装最新 CMake：

```bash
pip install --upgrade cmake
```

安装后版本：`cmake version 4.3.4`，路径：`~/.local/bin/cmake`（该路径已在 $PATH 中，覆盖了系统 `/usr/bin/cmake`）。

## 4. 自定义 autoware_msgs 包缺失（核心问题）

### 现象

```text
fatal error: autoware_msgs/msg/lane.hpp: No such file or directory
```

controller、path_generator、boundary_detector 等多个包依赖 `autoware_msgs` 中的 `Lane`、`Waypoint`、`Command` 消息类型，但系统中没有这个包。

### 根因

系统 apt 安装的 `ros-humble-autoware-msgs` 是 **Autoware.Auto 项目** 的消息包（含 `VehicleControlCommand`、`SteeringReport` 等），不含项目代码引用的 `Lane`、`Waypoint`、`Command`。项目需要的是 **FSD 自定义** 的 `autoware_msgs`。

### 修复

在 `/home/wuta/WUTA/WUTA-FSD/ros2_ws/src/common/autoware_msgs/` 下创建了自定义消息包，包含 3 个消息：

**`msg/Waypoint.msg`**
```text
geometry_msgs/PoseStamped pose
geometry_msgs/TwistStamped twist
```

**`msg/Lane.msg`**
```text
std_msgs/Header header
Waypoint[] waypoints
```

**`msg/Command.msg`**
```text
float64 speed        # 速度 (m/s)
float64 angle        # 转向角 (degrees, 正=左转)
int32 dv_state       # 驾驶状态 (4=normal, 6=emergency)
```

同时移除了冲突的系统包：

```bash
sudo apt remove ros-humble-autoware-msgs
```

这样 `colcon build` 只会找到工作空间内的 `autoware_msgs`，不会混淆。

### 现象

```text

'/home/wuta/WUTA/WUTA-FSD/ros2_ws/src/control/controller/launch/'

| `mapping/cone_map_builder` | `config/` |
| `perception/lidar_detection` | `config/` |

> **注意**: 目录目前是空的。运行前需放置对应的 `.yaml` 参数文件和 `.launch.py` 启动脚本，否则节点启动时会使用硬编码默认值或报错。

+   rclcpp sensor_msgs geometry_msgs nav_msgs visualization_msgs wuta_msgs pcl_conversions
```

## 7. HPL 库缺失

### 现象

```text
fatal error: hpl/hpl_param.hpp: No such file or directory
```

boundary_detector 内的 thirdparty pathplanning 算法库引用了 HPL（HRT-D 团队自研的参数管理/日志库）。

### 根因

HPL 库在 ROS 1 中作为独立包存在，ROS 2 版本尚未就绪。

### 修复

在 `planning/boundary_detector/include/hpl/` 下创建了两个极简替代头文件：

**`hpl/hpl_param.hpp`** — 哑参数节点，`declParam()` 为空操作：
```cpp
#pragma once
#include <string>
namespace hpl {
class ParamNode {
public:
  explicit ParamNode(const std::string& /*ns*/) {}
  template<typename T>
  void declParam(T& /*var*/, const std::string& /*name*/) {}
};
}
```

**`hpl/hpl_log.hpp`** — CLOG 宏打印到 stdout：
```cpp
#pragma once
#include <cstdio>
#define CLOG(fmt, ...) printf("[CLOG] " fmt "\n", ##__VA_ARGS__)
```

同时在 `boundary_detector/CMakeLists.txt` 中给 `pathplanning` 库加了 include 路径：

```diff
- target_include_directories(pathplanning PUBLIC thirdparty/pathplanning/include)
+ target_include_directories(pathplanning PUBLIC thirdparty/pathplanning/include include)
```

> **注意**: 这意味着 `hpl::ParamNode` 不会从 YAML 文件读取参数，GlobalVariables 的成员变量使用未初始化值（后续第 8 节补充了合理默认值）。

## 8. pathplanning 第三方库的 C++ 兼容性问题

### 8.1 GlobalVariables 缺少成员和方法

`GlobalVariables.cpp` 使用了 `Redundancy`、`StartPointRedundancy`、`PlugPointSize` 等成员，但 `GlobalVariables.h` 没有声明它们。`Path.cpp` 调用了 `GetStartPointRedundancy()` 但头文件中没有这个 getter。

**修复**: 在 `GlobalVariables.h` 中补充了 7 个 getter 方法和对应的 private 成员变量。

### 8.2 Path.cpp 缺少标准库头文件

`Path.cpp` 使用了 `cos()`、`sin()`（来自 `<cmath>`）和 `assert()`（来自 `<cassert>`）但没有 include。

**修复**: 在文件开头补加 `#include <cmath>` 和 `#include <cassert>`。

### 8.3 多个头文件只有前向声明

`PathSearch.h`、`Path.h`、`Tool.h` 中使用了 `Point2d`、`Vect`、`MidPoint`、`Triangle` 作为 `std::vector` 模板参数或值类型参数，但只有 `class Xxx;` 的前向声明，缺少完整的类定义头文件。

**修复**: 将前向声明替换为 `#include` 实际头文件：
- `PathSearch.h`: 加 `#include "pathplanning/structs/Point2d.h"`、`MidPoint.h`、`Vect.h`、`Triangle.h`
- `Path.h`: 加 `#include "pathplanning/structs/Point2d.h"`、`MidPoint.h`
- `Tool.h`: 加 `#include` 全部 4 个 structs 头文件 + `<memory>`

## 遗留注意事项

1. **config/ 和 launch/ 目录为空**: 运行节点前需要补充 YAML 配置和 launch 启动脚本
2. **HPL 库是哑实现**: `hpl::ParamNode::declParam()` 不读取 YAML，pathplanning 的参数使用代码中的默认值（未显式初始化，依赖编译器行为）
3. **Clash GUI 需要手动启动**: 已创建桌面快捷方式，或开机自行配置 systemd user service 自启 Clash for Windows
4. **CMake 4.x 来自 pip**: 系统包管理器不管理它，升级需手动 `pip install --upgrade cmake`
